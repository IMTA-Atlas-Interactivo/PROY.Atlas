Propuesta de proyecto

4 Planteamiento y Estrategia

Con el propósito de lograr el objetivo enunciado, se recurrirá a diversos instrumentos computacionales, los que atriculados para el desarrollo de la tarea y mediante el insumo de información adecuadamente estructurada, permitirán la presentación gráfica, geográfica y/o numérica de los resultados del CEMIE-Océano. A continuación se aborda con más detalle este tema.

4.1 Libros interactivos

En el objetivo se hace referencia a libros con contenidos gráficos y numéricos en forma de atlas interactivos, lo que es una especialización del concepto más general: libros digitales interactivos. El concepto de libros interactivos tiene su origen en los libros pop-up para niños, creados por Julian Wehr en la década de los 1940 (Movable Book Society 2002; Boehm and Ziegler 2002). Estos libros físicos, permiten a los niños interactuar con su contenido, mediante el accionar de las pestañas de papel o cartón, provistas en el libro para animar o dar movilidad a las ilustraciones. Más tarde, con el advenimiento de los libros digitales o electrónicos, los portales Web y los libros en línea (Reilly 2003), la idea de que los lectores pudieran interactuar con su contenido permeó de los libros físicos a los libros digitales en línea.

Definición: Aunque todavía no existe una definición formal de lo que es un libro digital interactivo, para los propósitos de la presente propuesta, se entenderá que es un documento digital, visible en línea o no, con la estructura de un libro y que permite a sus lectores ineractuar dinámicamente con su contenido.

De modo que, el proyecto que se propone aquí, está orientado a la creación de libros digitales interacivos, en línea, mediante alguna conexión a Internet, para presentar contenidos textuales, gráficos, geográficos y numéricos, con los que el usuario/lector podrá interactuar para cambiar dinámicamente su apariencia y/o extraer o componer información específica relevante.

4.2 Instrumentos computacionales

Para desarrollar las tareas planteadas, se requiere de un conjunto de elementos, básicamente de software, a los que aquí se denomina instrumentos computacionales. Estos elementos se articulan de tal manera que de manera coordinada, logren la construcción de uno o más libros digitales interactivos, que se ponen a disposición de un usuario o lector de los mismos. la Figura 4.1 muestra la arquitectura del arreglo de instrumentos computacionales.

FIGURA 4.1: Arquitectura de configuración

A continuación se da una breve descripción de los elementos de la arquitectura mostrada en la Figura 4.1.

  1. Productor de libros interactivos. Es el núcleo del sistema que tiene básicamente dos funciones: dar estructura y formato al contenido del libro, y proveer de los mecanismos necesarios para la interacción con el usuario. Para estas funciones se emplean los siguentes componentes:
    • Máquina de Formatos. Es el componente encargado de dar formato y estructura a los libros. Para el presente proyecto se usará R Markdown, que es un intérprete del lenguaje de formatos Markdown, con algunas adecuaciones para R (Xie, Allaire, and Grolemund 2018).
    • Máquina de interacciones. Es el componente encargado de proveer los mecanismos necesarios para la interacción con el usuario. Para el presente proyecto se usará Shiny Server, que es un diseñador de interfaces gráficas para el lenguaje R y que además, provee de servicios de páginas para la Web de las interfaces generadas con dicho producto (Resnizky 2015).
  2. Máquina computacional. Es el componente que se encarga de hacer todos los cálculos necesarios y que está detrás de las peticiones que le haga el Productor de libros interactivos. Para el presente proyecto se usará el Lenguaje de programación R, que es un lenguaje originalmente creado para cálculos estadísticos, pero que con el paso del tiempo y la incoporación de miles de paquetes que extienden su funcionalidad, se ha convertido en un lenguaje de propósito general (Santana and Mateos Farfán 2014).
  3. Paquetes de R para tareas específicas. Diversos paquetes, o bibliotecas del lenguaje R se usarán de acuerdo con los requerimientos que se vayan presentando. En seguida se menciona una muestra de los paquetes cuya utilidad para el proyecto ya se vislumbra:
    • R Leaflet. Es un paquete de R que permite la visualización de información en un contexto geográfico. Esto es, permite ver la información de manera gráfica proveyendo el contexto visual de mapas geográficos. Este paquete de R, está basado en la biblioteca de código abierto leaflet de JavaScript (Lovelace, Nowosad, and Muenchow 2019).
    • Raster. Es un paquete de R que permite la manipulación y visualización de imágenes raster. Mucha de la información que se genera en el proyecto CEMIE-Océano es de esta naturaleza; esto es, algún valor numérico asociado con cada elemento de una rejilla de pixels (Lovelace, Nowosad, and Muenchow 2019).
    • ncdf4. Es un paquete que permite la creación y manipulación de archivos de datos en el formato NetCDF (https://www.unidata.ucar.edu/software/netcdf/docs/index.html). Muchos de los modeladores modernos, por ejemplo ROMS y WRF, utilizan este formato como entrada y salida de su información.
  4. Fuentes de Información. Las fuentes de información del sistema son diversas: salidas de modeladores, compilaciones o calculos de información. En la sección 4.3 se trata con mayor profundidad este tema.
  5. Libro interactivo. El libro o libros interactivos son la salida del sistema. Desde un punto de vista simple, se puede considerar como un portal Web dinámico con la estructura de un libro, pero provisto de elementos que le permiten al usuario/lector interactuar con su contenido. Estos elementos pueden ser por ejemplo, menúes, botones, barras de deslizamiento, cursores, etc.
